# Документация Noted Terminal

> Структура документации следует **The Gold Standard v3.4** — стандарту для проектов, где работают вместе Человек и AI.

## 📂 Структура

### 🟢 Core (Обязательно)
Эти файлы критичны для понимания проекта:

- **`ai-context.md`** — 🔑 Точка входа для AI. Стек, правила, навигация, инструкции.
- **`architecture.md`** — 🏗️ Техническая архитектура (КАК). Детали реализации, IPC, стек.
- **`dev-guide.md`** — 👨‍💻 Гайд для разработчика. Установка, запуск, билд, команды.
- **`user-guide.md`** — 👤 Гайд для пользователя. Хоткеи, фичи, как пользоваться.
- **`troubleshooting/`** — 🔧 База решений проблем. Нумерованные файлы (001-, 002-).

### 🟡 Optional (По необходимости)
Эти файлы полезны для контекста:

- **`philosophy.md`** — 🧠 Философия решений (ПОЧЕМУ). Причины архитектурных выборов.
- **`dev-journal/`** — 📔 История разработки. Дневник изменений (YYYY-MM-DD формат).

### 🔴 Временные файлы
- **`tmp-*.md`** — Временные заметки, планы, миграции. Будут удалены после завершения.

## 🤖 Инструкции для AI

**AI должен читать:**
- ✅ Все Core файлы (ai-context.md, architecture.md, dev-guide.md, user-guide.md)
- ✅ Папку troubleshooting/
- ✅ philosophy.md (если нужно понять "Почему")

**AI должен игнорировать:**
- ❌ dev-journal/ (архив истории)
- ❌ tmp-*.md (временные файлы)
- ❌ node_modules/, build-resources/, assets/

## 📖 Разделение ответственности

| Вопрос | Файл |
|--------|------|
| Почему выбрали этот подход? | `philosophy.md` |
| Как устроена архитектура? | `architecture.md` |
| Как запустить проект? | `dev-guide.md` |
| Как пользоваться приложением? | `user-guide.md` |
| Что делать если ошибка X? | `troubleshooting/00X-*.md` |
| Какая история изменений? | `dev-journal/YYYY-MM-DD-*.md` |

## 🎯 Принципы

1. **Единая точка входа:** ai-context.md управляет всем контекстом.
2. **Разделение "Зачем" и "Как":** philosophy.md vs architecture.md.
3. **Масштабируемость:** Файл превращается в папку при росте (3+ экрана).
4. **Kebab-case:** Все файлы и папки в нижнем регистре через дефис.
5. **Минимализм:** Только необходимое, без мусора в контексте AI.

## 📝 Создание новых документов

### Troubleshooting
```
docs/troubleshooting/00X-problem-name.md
```
Нумерация 001, 002, 003... для хронологии.

### Dev Journal
```
docs/dev-journal/YYYY-MM-DD-topic-name.md
```
Формат даты ISO 8601 для сортировки.

### Временные файлы
```
docs/tmp-migration-status.md
docs/tmp-refactoring-plan.md
```
Префикс `tmp-` указывает что файл будет удален.

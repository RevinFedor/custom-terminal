  Текущая проблема

  При переходе пользователя на Dashboard (view === 'dashboard'), компонент 
<Workspace /> unmount'ится вместе со всеми Terminal компонентами. При 
возврате
  терминалы создаются заново.

  Что теряется:
  - Визуальный буфер xterm.js (история вывода терминала)
  - Позиция прокрутки

  Что НЕ теряется:
  - PTY процессы (живут в main process, не умирают)
  - Новый вывод от PTY приходит в новый xterm

  Почему не решается простым способом

  1. Нельзя держать <Dashboard /> и <Workspace /> одновременно в DOM через 
display: none — xterm.js ломает layout и синхронизацию при display: none
  2. visibility: hidden с position: absolute тоже вызывает unmount по 
какой-то причине (возможно React.memo)
  3. Поднять TerminalArea выше в иерархию — требует большой переделки 
архитектуры

  Что нужно сделать

  Использовать SerializeAddon из xterm.js:
  1. Перед unmount — сериализовать буфер терминала в строку и сохранить (в 
store или localStorage по tabId)
  2. После mount — восстановить буфер из сохраненной строки через 
term.write(serialized)

  Код сериализации уже частично есть:
  - src/renderer/components/Workspace/Terminal.tsx — уже импортирует 
SerializeAddon
  - serializeAddonRef.current — уже создается

  Будущее развитие (не делать сейчас)

  В будущем планируется полное восстановление сессий после перезапуска 
приложения:
  - Сохранять сериализованный буфер в SQLite/JSON
  - Восстанавливать при старте (без живого PTY — read-only история)
  - Аналогично текущей фиче AI Sessions (Trojan Horse), но для Visual Layer 
терминала

  Текущая задача — подготовить инфраструктуру сериализации для решения 
проблемы unmount/remount при навигации.

⏺ Отлично, я разобрался в архитектуре. Теперь понимаю проблему и решение.

⏺ Понимание проблемы:
  - <Workspace /> unmount'ится при переходе на Dashboard
  - xterm.js инстансы уничтожаются, теряется визуальный буфер
  - PTY процессы живут, новые данные буферизируются в pendingBuffer
  - Нужно сериализовать старый буфер перед unmount и восстановить после
  mount

  Решение:
  1. Хранить буферы в useWorkspaceStore (Map<tabId, serializedBuffer>)
  2. Сериализовать через SerializeAddon перед unmount
  3. Восстанавливать через term.write() после mount

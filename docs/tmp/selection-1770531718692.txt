⏺ Да, можно. Вот что у нас есть:

  В main.js:
  - terminals Map: tabId → ptyProcess (где ptyProcess.pid — это PID шелла, zsh/bash)
  - Каждый таб хранит commandType ('claude' / 'gemini' / etc.)

  Ключевой факт: Claude CLI запускается как дочерний процесс нашего шелла. Значит PPID Claude'а ===
  ptyProcess.pid одного из наших табов.

  Архитектурно самый чистый вариант:

  В IPC handler system:get-claude-processes мы уже получаем PID каждого Claude процесса. Можно добавить
  ps -eo pid,ppid,command и проверить PPID:

  Наши shell PIDs: terminals.values() → [pid1, pid2, pid3...]
  Claude process PPID === pid2? → это НАШ, tabId = X
  Claude process PPID не совпадает → внешний

  Тогда результат будет:
  { pid, cwd, startTime, command, tabId: "proj-tab-3" }  // наш
  { pid, cwd, startTime, command, tabId: null }            // внешний

  И на Dashboard две группы:
  - "In-App" — Claude запущенные из наших вкладок (можно даже кликнуть → переключиться на таб)
  - "External" — запущенные из iTerm/VS Code/etc.

  Хочешь реализовать так, или есть другие идеи?

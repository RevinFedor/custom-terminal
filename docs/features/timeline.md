# Feature: Claude History Timeline

## Intro
Таймлайн — это вертикальная панель справа от терминала, отображающая хронологию сообщений в текущей сессии Claude Code. Позволяет быстро визуализировать прогресс и перемещаться к конкретным точкам диалога.

## User Flow
1. Пользователь запускает `claude` в терминале.
2. Справа появляется полоса с точками. Каждая точка — сообщение пользователя.
3. При наведении на точку появляется всплывающее окно с превью текста промпта.
4. При клике на точку терминал скроллится к месту, где этот промпт был введен (через `searchAndScroll` — поиск по первым 40 символам текста).

## 2. Behavior Specs
- **Visibility:** Timeline отображается всегда, когда в текущем табе есть активная сессия Claude (`claudeSessionId`).
    - **Persistent Mount:** Timeline никогда не удаляется из DOM при переключении на Home, чтобы не вызывать скачков верстки. Он скрывается через проп `isVisible`, сохраняя свою ширину (24px).
    - **Active State:** Если процесс Claude запущен, Timeline имеет стандартную яркость.
    - **Inactive State:** Если Claude не запущен, Timeline переходит в "приглушенный" режим (фон `rgba(40,40,40,0.4)`, прозрачность `0.6`). Это позволяет просматривать историю даже после завершения работы агента.
- **Backtrace Filtering:** Отображение только активной ветки диалога. Сообщения, отменённые через Escape или Undo, скрываются из Timeline автоматически.
- **Auto-Refresh:** Timeline обновляется автоматически каждые **2 секунды**, чтобы отражать актуальное состояние сессии Claude. Обновление загружает только данные из JSONL — поиск по буферу терминала не выполняется (чтобы не сбрасывать пользовательское выделение текста).

### Визуальные маркеры и состояния точек
Помимо точек сообщений и линий-разделителей, Timeline использует динамические состояния для отображения контекста:

| Элемент | Визуал | Значение |
|---------|--------|----------|
| **Normal Dot** | Кружок 4px | Стандартное сообщение пользователя. |
| **Viewport Dash** | Черточка 10×3px | Сообщение **видимо** в текущем вьюпорте терминала. |
| **Continued Dot** | Оранжевый кружок | Системное сообщение восстановления ("This session is being continued..."). Недоступно для клика. |
| **Unreachable Zone** | Красный фон (0.06) | Сообщения, текст которых **удален** из буфера терминала (после `/compact`). |
| **Fork Marker** | Синий разделитель | Точка ветвления сессии. |
| **Plan Mode** | Зелёный разделитель | Граница "Clear Context". |
| **Compact** | Оранжевый разделитель | Граница `/compact` в логе. |

### Визуальный фидбек и навигация
- **Click Feedback:** При клике на точку мгновенно появляется вращающийся спиннер (`animate-spin`). 
- **Error Handling:** Если текст сообщения не найден в буфере (например, из-за очистки истории), на месте точки на 1.2с появляется красный крестик.
- **Tooltip Sync:** Тултип отображается вплотную к левому краю таймлайна с мягким белым свечением (`shadow-white`) для лучшей читаемости.
- **Auto-Refresh & Debounce:** Обновление данных происходит каждые 2с. Проверка видимости во вьюпорте (трансформация в черточку) выполняется при скролле с дебаунсом 100мс. Проверка доступности в полном буфере (красный фон) — с дебаунсом 500мс.

### Fork Markers (Snapshot UUIDs)
- **Snapshot UUIDs:** При форке сессии в БД сохраняется "снимок" всех UUID сообщений текущей ветки. Это гарантирует, что метка форка останется привязанной к конкретному сообщению даже при изменении истории.
- **Empty Snapshots:** Метка форка корректно отображается в самом начале списка, даже если на момент форка в сессии еще не было сообщений.
- **Каскадное наследование:** Дочерние сессии наследуют все метки форков своих родителей.

### Plan Mode Markers
- **Детекция:** Каждая timeline-запись содержит `sessionId`. Когда `sessionId` меняется между соседними записями и в этой позиции нет fork-маркера — это граница plan mode.
- **Почему не через `sessionBoundaries`:** При форке копируется JSONL-файл вместе с bridge-записями из оригинальной цепи. `resolveSessionChain` следует по этим мостам и может пропускать промежуточные сессии (см. `knowledge/ai-automation.md`, секция 3.1). Поэтому детекция через прямое сравнение `sessionId` в записях надёжнее.
- **Начало списка:** Если первая запись является child-сессией (есть в `sessionBoundaries`) и записей от parent-сессии нет — маркер появляется перед первой записью.
- **Ретроактивность:** Работает для всех ранее сохранённых сессий, т.к. `sessionId` уже есть в каждой JSONL-записи.

### Click-to-Scroll
- При клике на точку вызывается `terminalRegistry.searchAndScroll(tabId, searchText)` — поиск первых 40 символов промпта в буфере терминала.
- Предварительного кеширования позиций (marker binding) нет — поиск выполняется по требованию при клике. Это предотвращает сброс пользовательского выделения текста в терминале.
- Используется таймер 250ms для различения одинарного и двойного клика (двойной клик запускает Range Copy, см. ниже).
- **Search Robustness:** Из поискового запроса удаляются специальные символы (escape-последовательности) и системные уведомления (`<task-notification>`), попадающие в лог.

### Smart Tooltip
- При наведении на точку появляется превью сообщения с кнопкой "Expand" и "Copy".
- Использует "Invisible Bridge" для плавного наведения (см. `knowledge/ui-ux-stability.md`).
- Для сообщений типа `continued` тултип показывает статус "Context Overflow Recovery".

### Range Copy (копирование диапазона)
Позволяет скопировать в буфер обмена часть диалога (несколько сообщений подряд).

**Запуск режима выделения:**
- **Двойной клик** по точке, ИЛИ
- **ПКМ** → "Начать копирование"

**Работа в режиме выделения:**
- При наведении на другие точки — синий индикатор показывает выбранный диапазон.
- **Одинарный клик по другой точке** → копирует диапазон и показывает анимацию.
- **Одинарный клик по начальной точке** → отменяет выделение.
- **Клик вне таймлайна** → отменяет выделение.
- **ПКМ в режиме выделения** → "Отменить копирование".
- **Selection Persistence:** Выделение не пропадает при наведении на контекстное меню (Portal), благодаря использованию `selectionStartIdRef` и блокировке сброса `hoveredIndex` в режиме выбора.

### Rewind (Откатиться)
Автоматизация процесса возврата к выбранному сообщению через TUI-меню истории Claude Code.

**User Flow:**
1. **ПКМ по точке** → "Откатиться".
2. **Фаза 1 (Компакт):** Система копирует все сообщения от выбранного до конца сессии (`claude:copy-range`), отправляет их в Gemini с промптом "Rewind (Откат)" для получения краткого резюме.
3. **Фаза 2 (Откат):** Вызывается IPC `claude:open-history-menu`. Процесс имитирует `Esc`+`Esc` для открытия меню Claude, находит нужную запись по префиксу текста (Text-matching), навигирует стрелками и подтверждает выбор (`Enter`).
4. **Фаза 3 (Вставка):** После перезапуска Claude система очищает инпут (`Ctrl+C`), вставляет сжатый контекст через универсальный механизм `safePasteAndSubmit` (чанками < 1024 байт для обхода macOS TTYHOG) и отправляет его (`\r`). См. `knowledge/terminal-core.md`.

**Scroll Stabilization (Авто-скролл):**
После отката Claude TUI часто вызывает "прыжки" скролла к началу буфера из-за интенсивной перерисовки. 
- Система автоматически подписывается на событие `onScroll` терминала после подтверждения отката.
- Используется **debounce 200ms**: каждое изменение скролла сбрасывает таймер.
- Когда скролл стабилизируется (не меняется 200мс) → вызывается `scrollToBottom()`.
- Также предусмотрен safety-фоллбэк на 5 секунд.

**Навигация (Text-based matching):**
Для выбора записи в TUI-меню используется сопоставление текста сообщения. Это надежнее порядкового индекса, так как меню Claude может содержать системные или скрытые записи. В `main.js` записи меню сравниваются по префиксу (30 символов). Если точного совпадения нет, используется порядковый индекс с ограничением (clamp).

**Визуальная обратная связь (пульсация):**
Во время процесса точка и диапазон пульсируют (`animate-pulse`), отражая текущую фазу:
- **Янтарный:** Компактизация (Gemini работает).
- **Фиолетовый:** Навигация в TUI (Откат).
- **Зелёный (1.2с):** Успешная вставка и завершение.

**Три фазы визуальной обратной связи (Range Copy):**

| Фаза | Стейт | Визуал |
|-------|-------|--------|
| Выделение | `selectionStartId` | Статичный синий индикатор |
| Загрузка | `copyingRange` | Мигающий синий (`animate-pulse`) |
| Успех | `copiedRange` (800ms) | Зелёный flash с glow |

**IPC:** `claude:copy-range` — использует `resolveSessionChain` + Backtrace (тот же unified pipeline, что и `claude:get-timeline`). Это гарантирует корректную работу при копировании через границы Plan Mode.

### Context Menu (ПКМ)
- **Позиционирование:** Конкретное меню открывается справа-посередине от курсора (`x = cursor + 4px`, `y = cursor - height/2`). Высота меню вычисляется динамически на основе количества кнопок.
- **Auto-close:** Меню автоматически закрывается при уходе курсора с его области (`onMouseLeave`).
- **Portal Rule:** Кнопки в контекстном меню (Portal) обязательно используют `e.stopPropagation()` — см. `knowledge/ui-ux-stability.md`.
- **Button Status:** Во время отката кнопка "Откатиться" заменяется на статус выполнения ("Компакт...", "Откат...", "Вставка...") и блокируется.

## Code Map
- `Timeline.tsx`: UI компонент (полоса, точки, портал для тултипов, range copy).
- `main.js` (`claude:get-timeline`): Парсинг и фильтрация JSONL. Детекция типа `continued`.
- `terminalRegistry.ts`: Методы `searchAndScroll`, `getVisibleText` (чтение вьюпорта) и `getFullBufferText` (проверка reachability).

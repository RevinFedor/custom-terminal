# Feature: Claude History Timeline

## Intro
Таймлайн — это вертикальная панель справа от терминала, отображающая хронологию сообщений в текущей сессии Claude Code. Позволяет быстро визуализировать прогресс и перемещаться к конкретным точкам диалога.

## User Flow
1. Пользователь запускает `claude` в терминале.
2. Справа появляется полоса с точками. Каждая точка — сообщение пользователя.
3. При наведении на точку появляется всплывающее окно с превью текста промпта.
4. При клике на точку терминал скроллится к месту, где этот промпт был введен (через `searchAndScroll` — поиск по первым 40 символам текста).

## 2. Behavior Specs
- **Visibility:** Timeline отображается всегда, когда в текущем табе есть активная сессия Claude (`claudeSessionId`).
    - **Persistent Mount:** Timeline никогда не удаляется из DOM при переключении на Home, чтобы не вызывать скачков верстки. Он скрывается через проп `isVisible`, сохраняя свою ширину (24px).
    - **Active State:** Если процесс Claude запущен, Timeline имеет стандартную яркость.
    - **Inactive State:** Если Claude не запущен, Timeline переходит в "приглушенный" режим (фон `rgba(40,40,40,0.4)`, прозрачность `0.6`). Это позволяет просматривать историю даже после завершения работы агента.
- **Backtrace Filtering:** Отображение только активной ветки диалога. Сообщения, отменённые через Escape или Undo, скрываются из Timeline автоматически.
- **Auto-Refresh:** Timeline обновляется автоматически каждые **2 секунды**, чтобы отражать актуальное состояние сессии Claude. Обновление загружает только данные из JSONL — поиск по буферу терминала не выполняется (чтобы не сбрасывать пользовательское выделение текста).

### Визуальные маркеры (горизонтальные линии 12×2px)
Между точками сообщений отображаются цветные разделители для визуализации структуры сессии:

| Маркер | Цвет | Значение |
|--------|------|----------|
| **Fork** | `#3b82f6` (синий) | Точка ветвления. Сессия была форкнута через приложение. |
| **Plan Mode** | `#48968c` (зелёный) | Граница "Clear Context" / Plan Mode в Claude. |
| **Compact** | `#f59e0b` (оранжевый) | Граница `/compact`. Отображается как линия вместо точки. |

### Fork Markers
- **Snapshot UUIDs:** При форке сессии в БД сохраняется "снимок" всех UUID сообщений текущей ветки. Это гарантирует, что метка форка останется привязанной к конкретному сообщению даже при изменении истории.
- **Empty Snapshots:** Метка форка корректно отображается в самом начале списка, даже если на момент форка в сессии еще не было сообщений.
- **Каскадное наследование:** Дочерние сессии наследуют все метки форков своих родителей.

### Plan Mode Markers
- **Детекция:** Каждая timeline-запись содержит `sessionId`. Когда `sessionId` меняется между соседними записями и в этой позиции нет fork-маркера — это граница plan mode.
- **Почему не через `sessionBoundaries`:** При форке копируется JSONL-файл вместе с bridge-записями из оригинальной цепи. `resolveSessionChain` следует по этим мостам и может пропускать промежуточные сессии (см. `knowledge/ai-automation.md`, секция 3.1). Поэтому детекция через прямое сравнение `sessionId` в записях надёжнее.
- **Начало списка:** Если первая запись является child-сессией (есть в `sessionBoundaries`) и записей от parent-сессии нет — маркер появляется перед первой записью.
- **Ретроактивность:** Работает для всех ранее сохранённых сессий, т.к. `sessionId` уже есть в каждой JSONL-записи.

### Click-to-Scroll
- При клике на точку вызывается `terminalRegistry.searchAndScroll(tabId, searchText)` — поиск первых 40 символов промпта в буфере терминала.
- Предварительного кеширования позиций (marker binding) нет — поиск выполняется по требованию при клике. Это предотвращает сброс пользовательского выделения текста в терминале.
- Используется таймер 250ms для различения одинарного и двойного клика (двойной клик запускает Range Copy, см. ниже).

### Smart Tooltip
- При наведении на точку появляется превью сообщения с кнопкой "Expand" и "Copy".
- Использует "Invisible Bridge" для плавного наведения (см. `knowledge/ui-ux-stability.md`).

### Range Copy (копирование диапазона)
Позволяет скопировать в буфер обмена часть диалога (несколько сообщений подряд).

**Запуск режима выделения:**
- **Двойной клик** по точке, ИЛИ
- **ПКМ** → "Начать копирование"

**Работа в режиме выделения:**
- При наведении на другие точки — синий индикатор показывает выбранный диапазон.
- **Одинарный клик по другой точке** → копирует диапазон и показывает анимацию.
- **Одинарный клик по начальной точке** → отменяет выделение.
- **Клик вне таймлайна** → отменяет выделение.
- **ПКМ в режиме выделения** → "Отменить копирование".

**Три фазы визуальной обратной связи:**

| Фаза | Стейт | Визуал |
|-------|-------|--------|
| Выделение | `selectionStartId` | Статичный синий индикатор |
| Загрузка | `copyingRange` | Мигающий синий (`animate-pulse`) |
| Успех | `copiedRange` (800ms) | Зелёный flash с glow |

**IPC:** `claude:copy-range` — использует `resolveSessionChain` + Backtrace (тот же unified pipeline, что и `claude:get-timeline`). Это гарантирует корректную работу при копировании через границы Plan Mode.

### Context Menu (ПКМ)
- Позиционирование контекстного меню проверяет границы окна (`window.innerWidth`, `window.innerHeight`) и корректирует координаты, чтобы меню не выходило за пределы приложения.
- Кнопки в контекстном меню (Portal) обязательно используют `e.stopPropagation()` — см. `knowledge/ui-ux-stability.md` (секция Portal + handleClickOutside).

## Code Map
- `Timeline.tsx`: UI компонент (полоса, точки, портал для тултипов, range copy).
- `main.js` (`claude:get-timeline`): Парсинг и фильтрация JSONL. Возвращает `entries` (с `sessionId`), `sessionBoundaries`, `latestSessionId`.
- `main.js` (`claude:copy-range`): Экспорт диапазона сообщений. Использует `resolveSessionChain` + Backtrace.
- `main.js` (`claude:get-fork-markers`): Загрузка fork-маркеров из БД.
- `terminalRegistry.ts`: Метод `searchAndScroll` для click-to-scroll навигации.

# Feature: Terminal Scripts

## 1. Intro + User Flow
Кнопка **Scripts** в `ProjectToolbar` предоставляет быстрый доступ к npm-скриптам проекта без необходимости вручную вводить команды в терминал.

**Flow:**
Пользователь кликает на кнопку "Scripts" → Выпадает список команд из `package.json` текущей рабочей директории (CWD) активного терминала → При выборе команды в терминал отправляется `npm run [name]`.

## 2. Behavior Specs
- **Динамическое считывание:** Список скриптов обновляется при каждом открытии выпадающего списка (запрашивается актуальный CWD через `terminal:getCwd`).
- **Фильтрация приватных команд:** Скрипты, название которых начинается с подчёркивания (например, `_rebuild`), автоматически скрываются из UI.
- **Индикация отсутствия:** Если в текущей директории нет `package.json` или в нём нет секции `scripts`, отображается надпись "Scripts not found".
- **Авто-переключение:** При запуске скрипта UI автоматически переключается в режим терминала, если был открыт Dashboard или Home проекта.

## 3. Code Map
- **UI Компонент:** `src/renderer/components/Workspace/ProjectToolbar.tsx`
    - `fetchScripts`: Логика чтения файла через IPC `file:read`.
    - `handleRunScript`: Отправка команды через `terminal:input`.
- **IPC Handlers:**
    - `file:read`: Чтение контента с диска (Main process).
    - `terminal:getCwd`: Запрос текущего пути процесса `node-pty`.

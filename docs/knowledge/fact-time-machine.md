# Feature: Time Machine (Rollback)

## Intro
Система автоматического сохранения состояний Gemini и возможность мгновенного отката (Rollback) к любому предыдущему сообщению пользователя.

## Behavior Specs

### 1. Авто-снепшоты (Snapshots)
При каждом изменении файла сессии Gemini (что соответствует новому сообщению):
- Система считает количество `user` сообщений в JSON.
- Если количество сообщений увеличилось, создается полная копия JSON в `~/.minayu/history/<sessionId>/turn-XXX.json`.
- Каждая точка истории содержит полный контекст на тот момент.

### 2. Механизм отката (Rollback)
При выборе точки отката в UI:
1. **Cleanup:** Удаляются все снепшоты "будущего" (те, что старше выбранного терна). Это обеспечивает линейность истории.
2. **Restore:** Выбранный снепшот копируется поверх оригинального файла сессии Gemini.
3. **Restart:** Текущий терминал закрывается, и создается новый с автоматическим выполнением `gemini -r <sessionId>`.

### 3. UI Индикация
В InfoPanel отображается список "прыжков":
- Превью текста сообщения пользователя (первые 100 символов).
- Таймстемп.
- Кнопка ↺ для инициации отката.

## Code Map
- **Backend:** `src/main/main.js` — функции `saveGeminiHistorySnapshot`, `gemini:rollback`.
- **Watcher:** `src/main/main.js` — `gemini:start-history-watcher` отслеживает изменения в реальном времени.
- **UI:** `src/renderer/components/Workspace/panels/InfoPanel.tsx` — рендеринг списка тернов и логика вызова отката.

# ОПЫТ: Динамические стили и Tailwind v4

## Проблема
При попытке использовать динамические значения для стилей (например, цвет таба из базы данных) через классы Tailwind типа `bg-[${color}]` или `text-${color}`, стили не применяются в рантайме.

### Симптомы
Цвет вкладки, ширина панели или другие динамические параметры не меняются визуально, даже если данные в сторе обновились. Элемент остается со стилями по умолчанию, так как Tailwind не сгенерировал нужный CSS-класс для случайного HEX/RGB значения.

## Причина
Tailwind — это **статический** компилятор. Он сканирует ваш исходный код на наличие полных названий классов и включает в финальный CSS только их. Если класс формируется динамически (через конкатенацию строк), компилятор его не видит и не генерирует соответствующий CSS-код.

## Решение
Для значений, которые приходят из базы данных или вычисляются в рантайме (цвета табов, ширина панелей), необходимо использовать **Inline Styles** (атрибут `style` в React).

### Пример (ПРАВИЛЬНО):
```tsx
<div 
  className="w-4 h-4 rounded-full" 
  style={{ backgroundColor: tab.color || '#ccc' }} 
/>
```

### Пример (НЕПРАВИЛЬНО):
```tsx
<div className={`w-4 h-4 rounded-full bg-[${tab.color}]`} />
```

## Исключение
Если набор динамических значений ограничен и известен заранее, можно использовать `safelist` в конфигурации Tailwind или прописать полные названия классов в отдельном объекте-маппинге. Но для произвольных RGB/HEX цветов из БД inline styles — единственный надежный способ.

# Feature: Gemini Session Management

## Intro
Глубокая интеграция с Gemini CLI (0.25+). Обеспечивает захват сессий, их сохранение в базе данных и возможность полноценного ветвления (True Fork).

## Behavior Specs

### 1. Захват сессии (Gemini Sniper)
В отличие от Claude, Gemini создает файл сессии только после первого сообщения.
- **Watcher:** При вводе `gemini` активируется `fs.watch` на папку `~/.gemini/tmp/<hash>/chats/`.
- **Lifecycle:** Вочер не имеет короткого таймаута и живет до закрытия вкладки (или до обнаружения файла).
- **Detection:** При появлении файла `session-*.json` из него извлекается полный UUID (`sessionId`).
- **ID Structure:** Имя файла содержит короткий 8-символьный хеш (например, `01862a96`), который является первыми 8 символами полного UUID внутри JSON. Gemini CLI в команде `-r` (resume) официально принимает UUID.

### 2. Ручной ввод и валидация (InfoPanel)
Интерфейс поддерживает ручную привязку сессий через "Set ID".
- **Short ID Support:** Пользователь может вставить как полный UUID, так и короткий 8-символьный ID из имени файла (например, `01862a96`).
- **Resolution:** Если введен 8-символьный hex, система автоматически сканирует папку чатов текущего проекта, находит файл с подходящим префиксом и «разворачивает» его в полный UUID перед сохранением в store. Это позволяет легко копировать ID прямо из имен файлов в `~/.gemini/tmp/`.
- **Real-time Feedback:** Инпут в боковой панели валидирует ID на лету, подсвечивая рамку (зеленый/красный) и показывая превью первого сообщения в тултипе. См. `knowledge/fact-session-capture.md`.

### 3. True Fork (Ветвление)
Обычный `gemini -r <id>` в новой вкладке приводит к тому, что оба терминала пишут в один файл. Для разделения контекста реализован True Fork:
1. Main-процесс находит оригинальный JSON-файл сессии.
2. Создает копию с новым UUID и текущим временем.
3. Патчит внутреннее поле `sessionId` в JSON.
4. Запускает Gemini с новым UUID.

### 4. Флаги запуска (spawn-with-watcher)
Хендлер `gemini:spawn-with-watcher` поддерживает комбинации флагов:
- **`gemini`** — новая сессия (по умолчанию).
- **`gemini -r <id>`** — resume конкретной сессии по UUID. Используется при Fork и ручном возобновлении.
- **`gemini -r`** (`bareResume`) — resume последней по mtime сессии. Файл находится через `readdirSync` + сортировка.
- **`gemini -y`** (`yesMode`) — auto-approve без ручного подтверждения. Используется, когда промпт заранее подготовлен.
- **`gemini -y -r <id>`** (`resumeSessionId` + `yesMode`) — resume prefilled-сессии с auto-approve. Основной режим для **Update Docs** (см. `fact-docs-update.md`).

### 5. Ограничения
- **Exclusive Session:** В одной вкладке может быть активна только одна AI-сессия (либо Claude, либо Gemini). Запуск второй блокируется с выводом предупреждения в терминал.

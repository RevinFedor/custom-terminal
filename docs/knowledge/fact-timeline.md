# Feature: Claude History Timeline

## Intro
Таймлайн — это вертикальная панель справа от терминала, отображающая хронологию сообщений в текущей сессии Claude Code. Позволяет быстро визуализировать прогресс и перемещаться к конкретным точкам диалога.

- **User Flow**
1. Пользователь запускает `claude` в терминале.
2. Справа появляется полоса с точками. Каждая точка — сообщение пользователя.
3. При наведении на точку появляется всплывающее окно с превью текста промпта.
4. **Клик на точку (Контекстный):**
   - Если панель истории **закрыта**: терминал скроллится к месту промпта.
   - Если панель истории **открыта**: происходит переход к соответствующему сообщению внутри панели истории. Клик по самому терминалу в этом режиме игнорируется.
   - **Исключение (Compact/Continued):** Записи типов `compact` и `continued` (Context Overflow) не имеют физической позиции в терминале. Клик по их маркерам на Timeline работает **только** при открытой History Panel, выполняя скролл внутри ленты.

## 2. Behavior Specs
- **Visibility (Range-based):** Timeline отображается всегда, когда в текущем табе есть активная сессия AI.
    - **Logic:** Вместо поиска текста промпта во вьюпорте используется принцип «владения блоком». Каждое сообщение пользователя "владеет" диапазоном строк от своей позиции до начала следующего сообщения.
    - **Indicator:** Точка превращается в черточку (Viewport Dash), если вьюпорт терминала пересекается с диапазоном строк этого сообщения. Это гарантирует, что таймлайн всегда показывает активную точку, даже если пользователь скроллит длинный ответ AI, а сам промпт уехал за экран.
    - **Response-Only Indicator:** Если вьюпорт пересекает диапазон entry, но позиция самого промпта (`markerRow`) находится **выше** верхней границы вьюпорта — значит пользователь видит только ответ AI. В этом случае черточка отображается приглушённым серым (`rgba(255,255,255,0.35)`) без glow, вместо яркого белого (`rgba(255,255,255,0.75)` + glow). Размер черточки одинаковый (10px). Определение: `blockStart < viewport.top`. Работает для обоих движков (Claude OSC markers, Gemini text positions).
    - **Invisible Intent (Психология фокуса):** Приглушение индикатора необходимо, чтобы пользователь мгновенно понимал: на экране сейчас только результат работы ИИ, а сам вопрос находится выше. Это снижает когнитивную нагрузку при быстром скролле длинных диалогов.
    - **Shield Logic:** Маркеры сжатия (`compact`) служат приоритетной границей. Если сразу после сжатия происходит техническая смена ID сессии, система блокирует создание ложного `FORK` маркера. См. [`fix-compact-logic.md`](fix-compact-logic.md).
    - **Bounds:** Timeline и панель истории ограничены верхней границей терминала. Они не заходят в область табов проектов или терминалов.
    - **Tab Persistence:** Состояние панели истории (открыта/закрыта) и её размер сохраняются индивидуально для каждой вкладки терминала. При переключении вкладок состояние восстанавливается.
    - **Persistent Mount:** Timeline никогда не удаляется из DOM при переключении на Home, чтобы не вызывать скачков верстки. Он скрывается через проп `isVisible`, сохраняя свою ширину (32px).
    - **Active State:** Если процесс Claude/Gemini запущен, Timeline имеет стандартную яркость.
    - **Inactive State:** Если агент не запущен, Timeline переходит в "приглушенный" режим (фон `rgba(40,40,40,0.4)`, прозрачность `0.6`). Это позволяет просматривать историю даже после завершения работы агента.
- **Backtrace Filtering:** Отображение только активной ветки диалога. Сообщения, отменённые через Escape или Undo, скрываются из Timeline автоматически.
- **Auto-Refresh:** Timeline обновляется автоматически каждые **2 секунды**, чтобы отражать актуальное состояние сессии. Обновление загружает только данные из JSONL/JSON — поиск по буферу терминала для обновления самих точек не выполняется.
    - **Performance Guard (`isVisible`):** Поллинг данных и запросы к SQLite выполняются только тогда, когда Timeline реально виден пользователю. Это предотвращает лишнюю нагрузку на диск и БД.

### Sizing & Scrolling (Compact Mode)
В отличие от ранних версий, где Timeline пытался уместить все точки в доступную высоту экрана, текущая реализация использует **принудительный скролл**.
- **Проблема:** В длинных сессиях (50+ сообщений) точки «схлопывались» до высоты 4px, превращаясь в нечитаемую серую линию.
- **Решение:** Внедрена минимальная высота шага (20px для обычных сообщений, 8px для дочерних агентов). 
- **Эффект:** Если количество сообщений превышает вместимость экрана, Timeline начинает скроллиться. Это сохраняет кликабельность и визуальную разницу между типами сообщений независимо от длины диалога.

### Layout Constraints (The Flex-Height Trap)
Для корректной работы скролла внутри сложной flex-сетки воркспейса используется специальный CSS-хак.
- **Корневая причина:** Стандартный `height: 100%` на flex-child (скролл-контейнере) часто не резолвится Chromium, если родитель не имеет фиксированной высоты. Это приводило к тому, что контейнер Timeline растягивался на всю высоту контента вместо появления скроллбара.
- **Решение:** Использование `position: absolute; inset: 0` для внутреннего скролл-контейнера. Это гарантирует, что контейнер будет ограничен физическими размерами родительской 32px-полосы, заставляя `overflow-y: auto` работать корректно.

### Визуальный фидбек и навигация
- **Click Feedback:** При клике на точку мгновенно появляется вращающийся спиннер (`animate-spin`). 
- **Error Handling:** Если текст сообщения не найден в буфере (например, из-за очистки истории), на месте точки на 1.2с появляется красный крестик.
- **Tooltip Sync:** Тултип отображается вплотную к левому краю таймлайна с мягким белым свечением (`shadow-white`) для лучшей читаемости.
- **Auto-Refresh & Debounce:** Обновление данных происходит каждые 2с. Проверка видимости во вьюпорте (трансформация в черточку) выполняется при скролле с дебаунсом 100мс. Проверка доступности в полном буфере (красный фон) — с дебаунсом 500мс.
- **hasAnyPosition Guard:** Если `computePositions()` не нашёл ни одной позиции (все entries = -1), `checkReachability()` НЕ помечает entries как unreachable. Красный фон активируется только когда хотя бы одна entry имеет валидную позицию (т.е. остальные реально trimmed из scrollback). Это предотвращает полностью красный Timeline при свежем запуске с пустым буфером.
- **Boundary Gate Fix:** OSC 7777 injection в main.js гейтится на `claudeSpinnerBusy.has(tabId)` (не `bridgeKnownSessions`). См. [`fix-boundary-bridge-race.md`](fix-boundary-bridge-race.md).
- **Opaque Dots:** Все точки используют непрозрачные (solid) цвета. Это необходимо, чтобы они полностью перекрывали центральную вертикальную линию (Central Axis), исключая визуальный мусор и «просвечивание» полосы сквозь индикаторы.

### Кастомный скроллбар (In-strip Indicator)
Для Timeline реализован кастомный индикатор скролла, заменяющий системный скроллбар.
- **Почему не нативный:** Нативный скроллбар браузера (даже стилизованный через webkit) конфликтовал с расположенным рядом `Resizer`. Клик по скроллбару часто интерпретировался системой как начало ресайза боковой панели.
- **Реализация:** Тонкий (3px) ярко-белый индикатор внутри 32px-полосы Timeline.
- **Изоляция:** Индикатор имеет `pointer-events: none`. Пользователь скроллит Timeline вращением колеса мыши над его областью, а физические клики «пролетают» сквозь индикатор, не мешая работе соседнего ресайзера.

### Scroll-to-Bottom Arrow
Фиолетовая стрелка (`ChevronDown`, цвет `#a78bfa`) в нижней части Timeline.
- **Условие показа:** Появляется только когда ниже видимой области есть reachable (не-красные) entries. Проверка через `data-unreachable` атрибут на segment-div'ах.
- **Действие:** Плавный скролл (`behavior: 'smooth'`) scrollRef контейнера до конца.
- **Детекция:** Встроена в существующий `updateThumb()` scroll handler (без дополнительных event listeners). При `isAtBottom` (scrollTop >= maxScroll - 5) стрелка скрывается.
- **Motivation:** В длинных сессиях (50+ entries) пользователь может случайно проскроллить вверх и не знать, что ниже есть актуальные (не-красные) данные.

### Click-to-Scroll (Deterministic Navigation)

#### Claude Code (OSC Markers)
Для Claude Code навигация является детерминированной и работает через систему **PTY-маркеров**.
- **Механизм:** При клике система вызывает `terminalRegistry.scrollToEntry(uuid)`. Это мгновенный прыжок $O(1)$ к физической строке в буфере, зафиксированной в момент прихода данных от Claude.
- **Escape Carryover:** В PTY Middleware реализована защита от разрыва escape-последовательностей при вставке маркеров (PTY chunking). См. `knowledge/fix-osc7777-escape-carryover.md`.
- **Fallback:** Если маркер отсутствует (например, для самого первого сообщения сессии или после очистки скроллбека), система откатывается к поиску через `SearchAddon`.

#### Gemini CLI (Heuristic Search)
Поскольку Gemini работает в Alternate Buffer, маркеры xterm.js там не работают. Навигация использует многострочный поиск по тексту с алгоритмами повышения устойчивости:

- **Truncation Fallback:** Gemini TUI обрезает длинные сообщения в отображении (показывает только начало с префиксом `> `). Если multi-line поиск по полному тексту из JSON проваливается, система принимает match по одной первой строке при условии `firstLine.length >= 15`. Конкретная длина обрезки зависит от версии Gemini CLI и не зашита в коде.
- **Anchor Fallback:** Для предотвращения ложных срабатываний поиск каждой следующей записи начинается после позиции предыдущей (`startAfterRow`). Если поиск с якорем не дал результатов (якорь был ошибочным), система автоматически повторяет поиск по всему буферу.
- **Near-separators Filtering:** При генерации поисковых ключей (`getSearchLines`) система игнорирует не только чистые разделители (`────`), но и строки с "шумом" (например, `────╯`), где частота одного символа составляет $>80\%$. Это предотвращает использование декораций TUI как уникальных якорей.
- **Improved Thresholds:** Порог `isIsolatedShort` для коротких сообщений (типа "продолжай") увеличен с $+10$ до $+25$ символов, чтобы учитывать отступы и декорации, добавляемые Gemini TUI.

### Click-to-Scroll (Context-aware)
- При клике на точку вызывается `terminalRegistry.searchAndScrollToNth`.
- **Валидация контекста:** Поиск игнорирует вхождения текста внутри других сообщений. Скролл происходит только к строкам, которые начинаются с символов промпта (`❯`, `⏵`) или валидного отступа (для вставленного кода). Это синхронизирует логику клика с визуальным отображением точек.
- **Prompt Detection Fix:** Символ `>` исключен из списка промптов, так как он часто встречается в markdown-цитатах ответов Claude, вызывая ложные срабатывания.
- Предварительного кеширования позиций (marker binding) нет — поиск выполняется по требованию при клике.
- Используется таймер 250ms для различения одинарного и двойного клика.
- **Search Robustness (Viewport Matching):** Для поиска используется только первые **50 символов** сообщения (вместо 80). Это гарантирует, что ключ поиска не пересечет границу переноса строки (line wrap), который в Ink TUI часто случается раньше из-за отступов. Из поискового запроса удаляются специальные символы.

### Smart Tooltip
- При наведении на точку появляется превью сообщения с кнопкой "Expand" и "Copy".
- Использует "Invisible Bridge" для плавного наведения (см. `knowledge/fact-react-patterns.md`).
- Для сообщений типа `continued` тултип показывает статус "Context Overflow Recovery".

### Range Copy (копирование диапазона)
Позволяет скопировать в буфер обмена часть диалога (несколько сообщений подряд).

**Запуск режима выделения:**
- **Двойной клик** по точке, ИЛИ
- **ПКМ** → "Начать копирование"

**Работа в режиме выделения:**
- При наведении на другие точки — синий индикатор показывает выбранный диапазон.
- **Одинарный клик по другой точке** → копирует диапазон и показывает анимацию.
- **Одинарный клик по начальной точке** → отменяет выделение.
- **Клик вне таймлайна** → отменяет выделение.
- **ПКМ в режиме выделения** → "Отменить копирование".
- **Selection Persistence:** Выделение не пропадает при наведении на контекстное меню (Portal), благодаря использованию `selectionStartIdRef` и блокировке сброса `hoveredIndex` в режиме выбора.

### Rewind (Откатиться)
Автоматизация процесса возврата к выбранному сообщению через TUI-меню истории Claude Code.

**User Flow:**
1. **ПКМ по точке** → "Откатиться".
2. **Фаза 1 (Компакт):** Система копирует все сообщения от выбранного до конца сессии (`claude:copy-range`), отправляет их в Gemini с промптом "Rewind (Откат)" для получения краткого резюме.
3. **Фаза 2 (Откат):** Вызывается IPC `claude:open-history-menu`. Процесс имитирует `Esc`+`Esc` для открытия меню Claude, находит нужную запись по префиксу текста (Text-matching), навигирует стрелками и подтверждает выбор (`Enter`).
4. **Фаза 3 (Вставка):** После перезапуска Claude система очищает инпут (`Ctrl+C`), вставляет сжатый контекст через `safePasteAndSubmit` (программный paste: чанки < 1024 байт + sync marker verification + автоматический `\r`). Это **программная** операция — текст идёт напрямую в PTY, минуя xterm.js. См. `knowledge/fact-fact-terminal-core.md`.

**Scroll Stabilization (Авто-скролл):**
После отката Claude TUI часто вызывает "прыжки" скролла к началу буфера из-за интенсивной перерисовки. 
- Система автоматически подписывается на событие `onScroll` терминала после подтверждения отката.
- Используется **debounce 200ms**: каждое изменение скролла сбрасывает таймер.
- Когда скролл стабилизируется (не меняется 200мс) → вызывается `scrollToBottom()`.
- Также предусмотрен safety-фоллбэк на 5 секунд.

**Навигация (RGB Visual Search & Safety Guard):**
Для выбора записи в TUI-меню используется **Визуальный Поиск** с защитой.
- **Алгоритм:** Нажимает `Arrow Up` и анализирует сырой вывод PTY на предмет RGB-цвета выделения.
- **Нормализация:** Текст цели и текст на экране очищаются (`.trim()`) перед сравнением, чтобы игнорировать разницу в отступах.
- **Safety Guard:** Если после всех попыток текст не найден, операция **прерывается с ошибкой** (меню закрывается через `Escape`). Это предотвращает случайный выбор неверной записи (например, сброс в начало сессии).
- Детали реализации: `knowledge/fix-rewind-navigation.md`.

**Визуальная обратная связь (пульсация):**
Во время процесса точка и диапазон пульсируют (`animate-pulse`), отражая текущую фазу:
- **Янтарный:** Компактизация (Gemini работает).
- **Фиолетовый:** Навигация в TUI (Откат).
- **Зелёный (1.2с):** Успешная вставка и завершение.

**Три фазы визуальной обратной связи (Range Copy):**

| Фаза | Стейт | Визуал |
|-------|-------|--------|
| Выделение | `selectionStartId` | Статичный синий индикатор |
| Загрузка | `copyingRange` | Мигающий синий (`animate-pulse`) |
| Успех | `copiedRange` (800ms) | Зелёный flash с glow |

**IPC:** `claude:copy-range` — использует `resolveSessionChain` + Backtrace (тот же unified pipeline, что и `claude:get-timeline`). Это гарантирует корректную работу при копировании через границы Plan Mode.

### Context Menu (ПКМ)
- **Позиционирование:** Конкретное меню открывается справа-посередине от курсора (`x = cursor + 4px`, `y = cursor - height/2`). Высота меню вычисляется динамически на основе количества кнопок.
- **Auto-close:** Меню автоматически закрывается при уходе курсора с его области (`onMouseLeave`).
- **Portal Rule:** Кнопки в контекстном меню (Portal) обязательно используют `e.stopPropagation()` — см. `knowledge/fact-react-patterns.md`.
- **Button Status:** Во время отката кнопка "Откатиться" заменяется на статус выполнения ("Компакт...", "Откат...", "Вставка...") и блокируется.

### Sub-Agent Orchestration View (Gemini only)
При работе с оркестратором Gemini → Claude, Timeline поддерживает визуализацию работы суб-агентов.

**Phase 1 — Visual Differentiation:**
- Записи `[Claude Sub-Agent Response]` отображаются **индиго** цветом (`#818cf8`) вместо белого.
- Записи `[Claude Sub-Agent Timeout]` отображаются **красным** (`#ef4444`).
- Метаданные агента (имя, taskId) парсятся из footer ответа: `Tab: <name> | Task ID: <id> | Model: <model>`.
- **Footer Variants:** Footer может начинаться с `Tab:`, `Task ID:` или `Model:` (без `Tab:`/`Task ID:`). Если `Tab:` отсутствует, имя формируется из модели: `Claude (<model>)`.
- **Closing Tag Guard:** `parseSubAgentMeta()` требует наличие И открывающего `[Claude Sub-Agent Response]` И закрывающего `[/Claude Sub-Agent Response]`. Это предотвращает ложные срабатывания когда пользователь упоминает тег в тексте промпта.
- Backend: `parseSubAgentMeta()` в `gemini-data.js`.

**Phase 2 — Collapsible Groups:**
- **Раунды оркестрации:** ВСЕ последовательные sub-agent entries схлопываются в одну ноду (один "раунд оркестрации"), независимо от TaskID или имени агента.
- **Motivation:** В реальном workflow Gemini часто вызывает разных агентов последовательно (Research → Implementation → Test). Группировка всех ответов между двумя промптами пользователя в одну ноду значительно упрощает навигацию по Timeline, сокращая количество точек с 50+ до 10-15.
- **Collapsed:** одна индиго-точка с badge `×N` справа. Клик → expand.
- **Expanded:** toggle-бар сверху (горизонтальная индиго-линия 8×2px, клик → collapse) + ВСЕ entries группы как children (indent, уменьшенный dot 3px, кликабельные для навигации).
- **Dual-role header:** Первый entry группы (`isGroupHeader: true && isGroupChild: true`) — при collapsed работает как toggle, при expanded рендерится как child entry + отдельный toggle-бар сверху.
- **Tooltip Guard:** Badge `×N agents` в tooltip показывается только когда группа **collapsed** (`!expandedGroups.has(gi.groupId)`). При expanded каждый child entry показывает собственный tooltip без group badge.
- Группировка: `computeGroups()` возвращает `Map<number, GroupInfo>`. Собирает уникальные имена агентов (`Set<string>`). `agentName` = имя если один агент, `"N agents"` если несколько, `"Claude"` если unnamed.
- Visibility агрегируется для collapsed headers (any child visible → header visible).
- Состояние: `expandedGroups: Set<string>`, сбрасывается при смене сессии.

**Phase 3 — Tree View:**
- **Абсолютное наложение:** При активации режима дерева Timeline расширяется с 32px до 160px **влево**, используя `position: absolute`.
- **Motivation (Stability):** Использование абсолютного позиционирования поверх терминала (с `backdrop-filter: blur(8px)`) предотвращает Layout Shift. Терминал не меняет свои размеры, что критично для стабильности Canvas-рендерера и предотвращения лишних `SIGWINCH` сигналов в PTY (которые могли бы вызвать перерисовку TUI).
- **Labels:** В tree mode каждая нода показывает текстовый label (имя агента или превью промпта).
- **Визуал:** Вертикальные линии-коннекторы для sub-agent записей (CSS absolute positioned). Центральная ось перемещается влево (left: 4px).
- Tooltip `right` позиция учитывает ширину tree mode.
- Состояние: `treeMode: boolean`, персистится в `UIStore` (per-tab).
- **Trigger:** Кнопка переключения (иконка GitBranch) вынесена в `InfoPanel` (сайдбар) рядом с кнопкой "AI SESSION".

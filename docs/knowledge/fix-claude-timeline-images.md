# Fix: Claude Timeline Reachability for Image Messages

### Симптомы
- Точки на Timeline (история Claude) отображаются с **красным фоном** (unreachable), если сообщение содержит прикрепленное изображение.
- При этом клик по точке работает корректно (терминал скроллится к сообщению).
- Сообщение не подсвечивается «черточкой» (visibility), когда оно находится в области видимости.

---

## 1. Проблема: Несоответствие JSONL и TUI

### Что происходит
1. **JSONL:** Claude Code сохраняет полный текст сообщения и данные изображения в структурированном виде.
2. **TUI (Терминал):** При отправке сообщения с картинкой Claude Code выводит в терминал только `❯ [Image #1]`. Текст пользователя в строке промпта часто отсутствует.
3. **Timeline Search:** Система ищет в буфере терминала текст из JSONL (например, "Вот скриншот"). Не найдя его рядом с промптом, она считает сообщение удаленным из буфера.

### Почему клик работал
Клик использует **OSC 7777 маркеры** (инъекция в поток данных), которые привязаны к физической строке в буфере xterm.js и не зависят от текстового поиска.

---

## 2. Решение: Fallback на Image Placeholders

Внедрена двухуровневая детекция для сообщений с изображениями:

1. **Парсинг (`claude-data.js`):**
   - Улучшена детекция флага `hasImage`.
   - Если сообщение содержит только изображение (без текста), оно больше не игнорируется, а добавляется в Timeline с текстом `[Image]`.

2. **UI (`Timeline.tsx`):**
   - **Visibility:** Если у записи стоит флаг `hasImage`, система ищет в видимой области терминала паттерн `[Image #`. Если он найден — сообщение подсвечивается как видимое.
   - **Reachability:** Аналогично, при проверке наличия сообщения в полном буфере, паттерн `[Image #` используется как запасной ключ (fallback), если основной текст не найден.

## Результат
- Сообщения с картинками больше не окрашиваются в красный ошибочно.
- Индикатор видимости (трансформация точки в черточку) теперь корректно работает для таких сообщений.

# Fix: Claude Plan Mode Session Chain

## Проблема: Разрыв цепочки при Clear Context
С января 2026 года Claude Code CLI изменил поведение в режиме **Plan Mode**. При выборе опции **Clear Context** создается новый JSONL-файл сессии, который начинается с `parentUuid: null` и не содержит записи-мостика (`bridge entry`) с указанием ID предыдущей сессии. 

Это приводило к двум проблемам:
1. **Timeline** обнулялся: алгоритм backtrace останавливался на первой записи новой сессии, так как не видел связи с родителем.
2. **Export** терял контекст: невозможно было выгрузить полную историю диалога, включая сообщения до сброса контекста.

## Решение: SQLite Side-car Mapping
Поскольку файл сессии больше не является надежным источником связей, внедрена система внешнего маппинга через SQLite.

### 1. Перехват в реальном времени
StatusLine Bridge (`main.js`) отслеживает изменения файла `~/.claude/bridge/...` через `fs.watch`. Как только детектируется смена `sessionId` для активного таба, система мгновенно записывает связь в таблицу `session_links`.
- **Почему не JSONL:** Запись в БД происходит в момент детекции, что быстрее, чем Claude успевает сделать flush данных в JSONL на диск.

### 2. Патч алгоритма Backtrace
Алгоритм построения истории теперь имеет два пути перехода между сессиями:
- **Путь A (Legacy):** Через `_isBridge` запись внутри JSONL (если Claude её создал).
- **Путь B (Fallback):** Если достигнута вершина сессии (`parentUuid: null`), алгоритм обращается к `session_links` в SQLite. Если связь найдена, он «перепрыгивает» на последнюю запись родительской сессии.

## Особенности и ограничения
- **Race Condition:** В автотестах или при экстремально быстром переключении Timeline может не найти старый JSONL-файл в первые секунды после сброса контекста (Claude CLI еще не успел его создать или закрыть). В UI это решается естественной задержкой пользователя перед открытием истории.
- **Целостность:** Если база данных SQLite будет удалена, связь между сессиями, созданными через Plan Mode, будет потеряна (в отличие от обычных форков, которые сохраняют bridge-записи в JSONL).

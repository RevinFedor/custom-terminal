# Fix: Claude Visual Glitches (OSC 8 & Image Timeline)

### Симптомы
1. **Сломанная разметка:** В терминале Claude Code внезапно появляются длинные фиолетовые пути файлов (например, `~/.claude/projects/...`), которые должны быть скрыты. Разметка "едет", текст перекрывается.
2. **Красный Timeline:** Сообщения с прикрепленными изображениями отображаются с красным фоном (unreachable), хотя клик по ним работает.
3. **Отсутствие подсветки:** Сообщения с картинками не подсвечиваются «черточкой» при скролле.

---

## 1. Конфликт UUID Colorization и OSC 8

### Проблема (Invisible Intent)
Claude Code использует **OSC 8 гиперссылки** для маскировки внутренних путей и превью изображений:
`\x1b]8;;file://<PATH>\x07[Visible Text]\x1b]8;;\x07`

В приложении работает глобальный механизм `colorizeUUIDs()`, который находит UUID и красит их в фиолетовый.
**Ловушка:** Если UUID находится внутри URL в OSC-последовательности, простая замена через `.replace()` вставляет ANSI-коды (CSI) прямо в середину OSC. Это заставляет парсер `xterm.js` аварийно завершить обработку ссылки, из-за чего скрытый URL вываливается в терминал как видимый фиолетовый текст.

### Решение
Функция `colorizeUUIDs` переведена на **посимвольный парсинг** потока данных.
- Если обнаружен старт OSC (`\x1b]`), система ищет терминатор (`\x07` или `\x1b`) и пропускает весь блок без изменений.
- Колоризация UUID применяется только к обычному текстовому содержимому.

---

## 2. Специфика Image Placeholders в TUI

### Проблема: Несоответствие JSONL и Терминала
В файлах сессий (`.jsonl`) Claude Code хранит полный текст промпта. Однако в терминале (TUI) при наличии картинки он выводит только заглушку: `❯ [Image #1]`.
Стандартный механизм проверки «досягаемости» (Reachability) в Timeline ищет в буфере терминала текст из JSONL. Не найдя его рядом с символом промпта, система считала сообщение удаленным из скроллбека (красный фон).

### Решение: Двухуровневый Fallback
1. **Парсинг (`claude-data.js`):** При извлечении данных добавлен флаг `hasImage`. Сообщения, состоящие только из картинок, теперь получают текст `[Image]` вместо того, чтобы быть пропущенными.
2. **Timeline (`Timeline.tsx`):**
   - **Visibility:** При проверке видимости во вьюпорте, если у записи есть флаг `hasImage`, система дополнительно ищет строку `[Image #`.
   - **Reachability:** Аналогичный fallback добавлен в проверку полного буфера. Если текст промпта не найден, но в буфере есть `[Image #`, сообщение считается живым.

### Почему клик всегда работал
Навигация по Timeline (клик) использует **OSC 7777 маркеры**, которые инжектируются в поток в момент прихода данных и привязаны к физической строке. Они не зависят от текстового содержимого сообщения и работают даже если текст полностью не совпадает.

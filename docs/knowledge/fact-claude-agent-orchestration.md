# Feature: Claude Agent Orchestration (Gemini → Claude)

## Intro
Концепция «Терминал как оркестратор» (Agent-to-Agent без MCP) позволяет Gemini CLI делегировать задачи Claude Agent SDK напрямую через терминальный поток. Приложение выступает в роли «умного моста», перехватывающего инструкции и возвращающего результаты.

## Trigger Logic: Паттерн `:::claude`
Для активации оркестрации используется паттерн `:::claude <промпт> :::`.
- **Почему не `@`:** Изначально планировался префикс `@claude`, но он конфликтует с внутренним синтаксисом Gemini CLI (вызов инструментов типа `@ReadManyFiles`).
- **Синтаксис:**
    - `:::claude <prompt> :::` — отправить задачу в текущую сессию.
    - `:::claude:new <prompt> :::` — принудительно создать новую сессию.
    - `:::claude:status :::` — получить текущие метаданные сессии (токены, стоимость) без отправки промпта.

## Invisible Intent: Критические защиты (5 уровней)

### 0. geminiWatchers Guard
Детектор работает **только** в табах с активным Gemini watcher (`geminiWatchers.has(tabId)`). Это исключает ложные срабатывания в Claude-табах, shell-табах и любых других контекстах.

### 1. Синхронная блокировка (Race Condition)
**Проблема:** Импорт Claude Agent SDK асинхронен (`await import`). Пока SDK загружается, PTY продолжает генерировать события `onData`, что приводило к 10+ одновременным запускам агента на один и тот же паттерн.
**Решение:** Синхронная установка флага `cooldown` и очистка буфера ДО вызова асинхронного обработчика.

### 2. Spinner Guard (Ink TUI Re-render)
**Проблема:** Gemini CLI использует Ink (React TUI), который перерисовывает весь экран (включая историю диалога) при каждом движении спиннера. Это вызывало ложные срабатывания детектора на старых примерах паттерна в истории.
**Решение:** Если в чанке PTY обнаружены символы спиннера (`⠋⠙⠹...`), буфер детектора мгновенно сбрасывается. Мы считаем, что во время «размышления» Gemini не может выдавать свежие команды.

### 3. Механизм Arming
Детектор находится в состоянии `UNARMED` при старте. Он «взводится» (`ARMED`) только после первого ручного ввода пользователя (`terminal:input`). Это гарантирует, что детектор не сработает на «переигрывание» (replay) истории терминала при восстановлении вкладок или форке.

### 4. User Input Suppression
Если пользователь вставляет текст, содержащий `:::claude` (например, объясняет Gemini, как пользоваться паттерном), устанавливается 5-секундный cooldown в `terminal:input` handler. Это предотвращает детекцию PTY-эхо пользовательского ввода.

### 5. Компактный буфер (4KB)
Буфер детектора ограничен 4KB (вместо 50KB). Старый re-rendered контент быстро вытесняется новыми данными, уменьшая окно для ложных срабатываний.

## Техническая реализация
- **Движок:** Claude Agent SDK V2 (программное управление вместо CLI).
- **Session Persistence:** Сессии Claude привязаны к ID вкладок терминала. Контекст сохраняется между вызовами.
- **Output Routing:** Ответ Claude оборачивается в маркеры `[Claude Agent Response]...[/]` и вставляется в Gemini через `safePasteAndSubmit` с 10-секундным защитным cooldown.

## UI Индикация
- **Banner:** Компактный индиго-бейдж «Claude Agent» в правом верхнем углу терминала (absolute top-2 right-2). Видим только при `claudeAgentStatus === 'running'`.
- **Toasts:** Уведомления о начале работы, успешной вставке ответа или ошибках.
- **Metadata:** В ответ включаются данные о номере хода (turn), расходе токенов и стоимости в USD.

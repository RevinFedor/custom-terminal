# AI Audit: API Call Logging

## Intro
Система аудита API-вызовов обеспечивает прозрачность "невидимых" операций, таких как суммаризация контекста при адоптации агентов (`adoptClaudeAgent`) или синхронный анализ документации (`update_docs`).

## Motivation: Почему отдельный лог?
В отличие от обычных чатов (`ai_sessions`), которые представляют собой интерактивный диалог, системные вызовы являются техническими транзакциями.
- **Потеря контекста:** Раньше результаты `adopt` существовали только в памяти и PTY-потоке. Если пользователь проскроллил терминал, информация о том, что именно Gemini узнал о Claude, была потеряна.
- **Контроль расходов:** Лог позволяет отслеживать потребление токенов и стоимость разовых операций, которые часто бывают тяжелыми (до 200K+ токенов за один `update_docs`).
- **Отладка:** Возможность увидеть точный payload и ответ API без необходимости копаться в системных логах.

## Data Schema (`api_call_log`)
Лог хранится в SQLite и включает:
- `call_type`: тип операции (`adopt`, `update_docs`, `research`).
- `payload_size`: объем переданных данных в байтах (позволяет оценить плотность контекста).
- `input_tokens` / `output_tokens`: точные метрики от провайдера.
- `result_text`: финальный результат, который был вставлен в чат или терминал.
- `session_meta` (новое, 2026-03): Облегченный снимок сессии в формате JSON, содержащий:
  - `iterations`: количество раундов в сессии
  - `compacts`: количество проведенных сжатий
  - `lastPrompts`: массив текстов последних 2-3 промптов (обрезаны до 200 символов) — достаточно для понимания контекста без перегрузки БД.
  - **Invisible Intent:** Переход от сохранения полного `input_payload` к `session_meta` позволяет создавать "легкие снимки" сессий для аудита и анализа контекста без существенного увеличения размера базы данных SQLite.

## UI Integration
История вызовов отображается во вкладке **AI Sessions** (NotesPanel) под списком активных чатов.
- **Real-time Sync:** UI слушает IPC-событие `mcp:agent-adopted` для мгновенного обновления после завершения адоптации.
- **Loading State:** В момент активного вызова в списке отображается анимированный лоадер, подтверждающий, что Main-процесс ожидает ответа от LLM.
- **Mini-Timeline Visualization:** Каждая запись в истории отображает компактный граф-снимок сессии:
  - **Зеленые блоки:** Раунды диалога (каждый блок = один ход в сессии).
  - **Фиолетовые блоки:** Сжатия (compact операции). Количество блоков отражает `iterations` и `compacts` из `session_meta`.
  - **Tooltip:** При наведении на график показываются точные числа итераций и компактов, плюс превью `lastPrompts` для быстрого контекста.
- **Persistence:** Лог сохраняется навсегда (с лимитом отображения последних 50 записей), обеспечивая "шрамы" разработки даже после перезапуска приложения.

# ОПЫТ: Навигация по JSONL через Backtrace

## Проблема: "Мёртвые" ветки диалога
Файлы `.jsonl`, которые генерирует Claude Code, являются журналом событий (append-only log), а не готовым списком сообщений. 
Когда пользователь нажимает **Escape (Undo)**:
1. Записи в файле НЕ удаляются.
2. В UI Claude откатывается к предыдущему состоянию.
3. Новое сообщение создаёт "вилку" (новую ветку) от старого родителя.

Если читать файл просто сверху вниз (как массив), Timeline покажет и отменённые сообщения, и новые, что ломает логику.

## Решение: Алгоритм Обратной Трассировки (Backtrace)
Вместо чтения подряд, мы собираем историю "с конца":

1. **Индексация:** Читаем весь файл и создаем Map `UUID -> Entry`.
2. **Определение головы:** Берем самую последнюю запись в файле (это текущее состояние).
3. **Обход:** Идем от "головы" назад к корню, используя поле `parentUuid`.
4. **Компакция:** Если встречаем маркер компакции, переходим по `logicalParentUuid`.
5. **Результат:** Инвертируем собранный список.

### Схема
```
A (Привет) ←── B (Здарова)  [Undo нажато]
↑
└── C (Hello)  <-- Голова (Текущее состояние)
```
Алгоритм выберет путь **A -> C**, полностью игнорируя **B**, так как на него никто не ссылается.

## Технические детали
- Логика реализована в `main.js` (IPC handler `claude:get-timeline`).
- Ветка фильтруется от служебных сообщений (`isSidechain: true`) и результатов инструментов (`tool_result`).

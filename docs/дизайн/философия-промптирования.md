# Философия проектирования промптов (Prompt Design)

Этот документ описывает принципы работы с текстовыми вставками (Snippets) и AI-промптами в **Noted Terminal**.

## Основная концепция: Проектирование через сборку

В отличие от традиционных систем, где промпт — это монолитный текст, наша философия строится на **модульности**. Промпт — это атомарный строительный блок (кирпичик), из которого пользователь-архитектор собирает сложную инструкцию.

### 1. Модульность и атомарность (Snippets)
Каждый пункт в меню `Insert Prompt` — это не просто текст, а специализированный инструмент:
- **docs_search**: Активация MCP-поиска по базе знаний.
- **ultrathink**: Перевод модели в режим глубоких рассуждений.
- **опиши user flow**: Шаблон для декомпозиции UX-сценариев.

*Принцип:* Промпт должен решать одну конкретную задачу. Если задача сложная — используй несколько промптов последовательно.

### 2. Группировка по интентам (Intent-based Grouping)
Все текстовые вставки распределены по группам в настройках (`AI` tab). Группировка отражает контекст использования:
- **`база`**: Фундаментальные инструменты управления (поиск, анализ, резюме).
- **`meta`**: Инструкции высокого уровня о том, *как* AI должен обрабатывать информацию (стиль, формат, ограничения).
- **`объясни`**: Шаблоны для разбора кода, логики или архитектурных решений.
- **`edge cases`**: Инструменты для поиска пограничных состояний и потенциальных багов.

### 3. Иерархия и "Вложенность"
Хотя в интерфейсе меню сейчас реализован один уровень вложенности (`Группа > Промпт`), концепция подразумевает более глубокую связь:
- **"Один промпт внутри другого"**: Это паттерн, при котором один сниппет подготавливает контекст для следующего. Например, сначала вставляется `база > docs_search`, а следом `meta > сначала обсуди`.
- **Контекстная зависимость**: Промпты могут ссылаться на глобальные правила из `Global-Templates`, которые инжектятся через систему трансклюзии (transclusion).

### 4. Управление через настройки
Вся структура (порядок групп, наличие промптов в меню, их содержание) полностью кастомизируется пользователем:
- **Drag-and-drop**: Позволяет выстраивать визуальный приоритет. Те инструменты, что нужны чаще, поднимаются выше.
- **Database-first**: Настройки хранятся в БД, что позволяет легко расширять список "базовых" инструментов без изменения кода.

### 5. Интеграция с Knowledge Base
Многие промпты из меню являются "ключами" для систем автоматизации:
- Вставка текста `docs_search` — это прямой сигнал для `semantic-router` или `docs_search MCP` начать поиск.
- Это превращает простое текстовое меню в **пульт управления агентом**.

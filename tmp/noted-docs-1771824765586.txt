# Plan: Migrate HistoryPanel from react-virtuoso to virtua

## Context
HistoryPanel uses react-virtuoso with `defaultItemHeight={150}` for virtual scroll. Real entry heights range 30px–800px. When scrolling up, Virtuoso measures real heights → scrollHeight changes → scroll position jumps (jitter). Deep research confirmed virtua 0.48.x has jump compensation built into its core Scroller module and is the best fit for chat UIs.

## Constraints
1. **`margin: '4px 0'` on HistoryEntry (lines 94, 106, 127)** — ResizeObserver's `contentRect` excludes margins → virtua undercounts height by 8px → gaps/drift. Must remove margins, use wrapper padding instead.
2. **No `atBottomStateChange` in virtua** — must manually compute from `onScroll` using `scrollSize - (offset + viewportSize) < 150`.
3. **No `initialTopMostItemIndex` in virtua** — use CSS flexbox bottom-anchor pattern (flexGrow spacer) + `scrollToIndex` on first load.
4. **react-virtuoso must NOT be uninstalled** — `ChatArea.tsx` still imports it.
5. **`useCallback` stale closure** (CLAUDE.md) — `loadHistory` deps `[sessionId, cwd]` are stable; internal scroll reads from refs only. Safe.
6. **Console interceptor** — use `[HP]` prefix for debug logs.

## Steps

### 1. Install virtua
```bash
npm install virtua@^0.48.6
```

### 2. Fix HistoryEntry margins
Remove `margin: '4px 0'` from 3 entry types (continued line 94, user line 106, assistant line 127). Spacing will be handled by wrapper `<div style={{ paddingTop: 4, paddingBottom: 4 }}>` inside virtualizer children.

### 3. Replace imports & ref type
```tsx
// Before
import { Virtuoso, VirtuosoHandle } from 'react-virtuoso';
// After
import { Virtualizer, VirtualizerHandle } from 'virtua';
```
Rename `virtuosoRef` → `virtualizerRef`, type `VirtualizerHandle`.

### 4. Remove `ready` state
Delete `const [ready, setReady] = useState(false)` and all references (`setReady(true)`, `setReady(false)`). Not needed with flexbox bottom-anchor.

### 5. Add manual at-bottom detection
```tsx
const handleVirtualScroll = (offset: number) => {
  if (!virtualizerRef.current) return;
  const { scrollSize, viewportSize } = virtualizerRef.current;
  isAtBottomRef.current = scrollSize - (offset + viewportSize) < 150;
};
```

### 6. Update scrollToIndex calls to virtua API
```tsx
// virtua API: scrollToIndex(index, { align: 'end' })
virtualizerRef.current?.scrollToIndex(newEntries.length - 1, { align: 'end' });
```

### 7. Replace Virtuoso JSX with flexbox + Virtualizer
```tsx
<div style={{ height: '100%', overflowY: 'auto', display: 'flex', flexDirection: 'column' }}>
  <div style={{ flexGrow: 1 }} />
  <Virtualizer ref={virtualizerRef} onScroll={handleVirtualScroll}>
    {entries.map((entry) => (
      <div key={entry.uuid} style={{ paddingTop: 4, paddingBottom: 4 }}>
        <HistoryEntry entry={entry} />
      </div>
    ))}
  </Virtualizer>
</div>
```

### 8. Initial scroll-to-bottom
In `loadHistory` initial path, after `setEntries` + `setLoading(false)`:
```tsx
requestAnimationFrame(() => {
  virtualizerRef.current?.scrollToIndex(newEntries.length - 1, { align: 'end' });
});
```

## Как это обычно решается?

### Вариант 1: Fixed heights (не подходит нам)
Если бы все entries были одинаковой высоты (например, 100px каждый), jitter был бы невидим.
**Нас не подходит**: assistants responses с markdown могут быть от 40px до 1000px.

### Вариант 2: Pre-measure на backend 
Перед отправкой в renderer, на main process посчитать **реальную высоту** каждого entry (отрендерить в offscreen canvas, измерить, потом выкинуть).
**Плюс**: Virtuoso будет знать точные высоты сразу → no jitter
**Минус**: 200ms задержка при загрузке, нужно учитывать CSS шрифт/цвет

### Вариант 3: Lazy progressive measurement
Рендерить entries по мере скролла, но **измеренные высоты сохранять** в React state.
```typescript
const [heightCache, setHeightCache] = useState<Map<uuid, number>>(new Map())

// После рендера каждого entry, измерить его высоту
useLayoutEffect(() => {
  if (ref.current) {
    const h = ref.current.offsetHeight;
    setHeightCache(prev => new Map(prev).set(uuid, h));
  }
}, [uuid])

// Передать Virtuoso:
<Virtuoso
  itemSize={(idx) => heightCache.get(entries[idx].uuid) || 150}
/>
```
**Плюс**: Асимптотически приближается к идеальности, скролл stabilizes по мере использования
**Минус**: Первый скролл всё равно jittery, нужен fallback

`defaultItemHeight={150}` — это **волшебное число** которое угадывает высоту. Когда угадка неправильна — jitter.

---

## Почему нейронка написала именно это?

Потому что в 2024-2025 году это **стандартный паттерн** для Virtuoso:
1. Установить `defaultItemHeight` 
2. Установить `increaseViewportBy` побольше
3. Надеяться что jitter будет мало

Это **работает** когда элементы примерно одного размера. Но для вашего случая (от 30px до 800px) этого недостаточно.

> ### 2. handleUpdateDocs - Full Flow
> 
> **Location:** Lines 293-436 in ActionsPanel.tsx
> 
> The function signature is: `handleUpdateDocs(source: 'session' | 'selection' | 'clipboard', e?: React.MouseEvent)`
> 
> **Step-by-step flow:**
> 
> 1. **Validation** -- checks for `activeTabId`, `activeProjectId`, and selection if source is 'selection'
> 2. **Content acquisition** (based on source):
>    - `'selection'`: Uses `terminalSelection` directly from store
>    - `'clipboard'`: Reads from `navigator.clipboard.readText()`
>    - `'session'`: Exports from JSONL/JSON files via IPC:
>      - For Gemini tabs: `ipcRenderer.invoke('gemini:copy-range', {...})`
>      - For Claude tabs: `ipcRenderer.invoke('claude:export-clean-session', {...})` with filter flags
>      - Multi-select: iterates all selected tabs, joins results with `========` separator
> 3. **Documentation prompt loading**:
>    - If `docPrompt.useFile` is true: reads from file via `ipcRenderer.invoke('docs:read-prompt-file', { filePath })`
>    - Otherwise: uses `docPrompt.inlineContent` inline string
> 4. **Create Gemini tab** -- names it `docs-gemini-01`, `docs-gemini-02`, etc. with `color: 'gemini'`
> 5. **Create pre-filled Gemini session** -- `ipcRenderer.invoke('gemini:create-prefilled-session', { sessionContent, systemPrompt, additionalPrompt, cwd })`
>    - This bypasses Gemini's `@file` truncation (~96KB) and `read_file` limit (5000 lines)
>    - Content goes directly into the session JSON's `content[]` array (what the model sees)
>    - `displayContent[]` shows a short summary in the TUI
>    - Also injects all `docs/knowledge/*.md` files into the context
> 6. **Spawn Gemini with resume** -- `ipcRenderer.send('gemini:spawn-with-watcher', { tabId, cwd, resumeSessionId })`
>    - This writes `gemini -r <uuid>\r` to PTY
> 7. **Wait for Gemini ready** -- `waitForGeminiReady()` listens for `[INSERT]` in terminal output (with 30s timeout)
> 8. **Trigger response** -- After 500ms delay, pastes `'Ответь на промпт выше.'` via `terminal:paste` with `submit: true`
> 
> **Cancellation** is checked at every async boundary via `cancelledRef.current`. The cancel handler (`handleCancelUpdateDocs`) sets the flag, closes the created Gemini tab if any, and resets state.

## 1. Архитектура Update Docs

**Есть 3 функции-хэндлера**, и у них **дублируется логика экспорта сессии**:

| Кнопка | Функция | Что делает |
|--------|---------|-----------|
| Заголовок "Update Docs" | `handleUpdateDocs('session')` | Export → temp file → spawn Gemini → paste prompt |
| ✂️ Ножницы | `handleUpdateDocs('selection')` | Selection → temp file → spawn Gemini → paste |
| 📋 Планшет | `handleUpdateDocs('clipboard')` | Clipboard → temp file → spawn Gemini → paste |
| 📄 Copy | `handleCopyDocsToClipboard()` | Export → assemble prompt → clipboard |
| [api] | `handleApiDocsRequest()` | Export → assemble prompt → Claude API → clipboard |

**Ответ на вопрос**: `gemini:spawn-with-watcher` вызывается **в одном месте** — внутри `handleUpdateDocs` (строка ~514). Менять запуск Gemini (с `-r` на `-y`) нужно было в двух файлах:
- `ActionsPanel.tsx` — передаём `yesMode: true` вместо `resumeSessionId`
- `main.js` — добавили ветку `else if (yesMode) { term.write('gemini -y\r') }`

Но сам вызов один, так что архитектура тут **корректная**.

**Проблема** — экспорт сессии (цикл по табам с `gemini:copy-range` / `claude:export-clean-session`) **продублирован 3 раза** почти идентично в `handleCopyDocsToClipboard`, `handleApiDocsRequest`, `handleUpdateDocs`. Это просится в одну общую функцию типа `exportSessionContent(tabs)`. Но это рефакторинг, не баг.

## 2. main.js — 5790 строк

**Факт**: файл `main.js` — это **монолит**. 5790 строк в одном файле. Docs-related хэндлеры занимают строки ~2800–2975 (3 хэндлера, ~170 строк).

**Корректно ли что docs-логика там?** — Да, это **правильное место**. `docs:api-request` делает `fetch` из main process специально для обхода CORS. `docs:save-temp` пишет файл на диск. Это задачи main process, renderer их делать не может/не должен.

**Удобно ли мне работать с 5790 строк?** — Честно:
- **Читать** — нормально, grep/search работают
- **Редактировать** — рискованно. Чем больше файл, тем выше шанс что `Edit` попадёт не в то место или что контекст не влезет. На 5800 строк это уже ощутимо
- **Понимать зависимости** — сложно. Общие переменные (tabs, ptyProcesses, geminiCommandQueue) разбросаны по всему файлу, непонятно кто от чего зависит

**Стоит ли рефакторить?** — Зависит от приоритетов. Вот трезвая оценка:

| За рефакторинг | Против |
|---|---|
| Легче находить баги | Работает и так |
| Меньше риск при редактировании | Рефакторинг main process — опасная операция |
| Логические модули (claude, gemini, docs, terminal) | Общее состояние (tabs, pty) усложняет разделение |
| Мне проще работать с файлами <1000 строк | Время на рефакторинг vs время на фичи |

Если решишь рефакторить, естественное разделение:
- `ipc/terminal.js` — PTY, paste, tabs
- `ipc/claude.js` — session export, timeline, fork
- `ipc/gemini.js` — spawn, rewind, copy-range
- `ipc/docs.js` — save-temp, api-request, read-prompt
- `main.js` — app lifecycle, window, shared state

Но это серьёзная задача, не на один вечер.

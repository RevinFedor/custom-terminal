       ipcMain.on('gemini:send-command', (event, tabId, command) => {
         console.log('[gemini:send-command] Received: tabId=' + tabId + ' command="' + command + '"');
         const term = terminals.get(tabId);
         if (!term) { console.log('[gemini:send-command] Terminal not found'); return; }

         const prev = geminiCommandQueue.get(tabId) || Promise.resolve();
         const next = prev.then(async () => {
           await drainPtyData(term, 300);
           term.write('\x01'); // Ctrl+A (beginning of line)
           term.write('\x0b'); // Ctrl+K (kill to end of line)
           await new Promise(r => setTimeout(r, 100));
           await safePasteAndSubmit(term, command, {
             submit: true,
             ctrlCFirst: false,
             fast: true,
             logPrefix: '[gemini:send-command:' + tabId + ']'
           });
           await drainPtyData(term, 500);
         });
         geminiCommandQueue.set(tabId, next.catch((err) => {
           console.error('[gemini:send-command] Error:', err.message);
         }));
       });

<thinking>
The issues are:

1. `scrollIntoView({ behavior: 'smooth', block: 'center' })` - smooth scrolling is too slow for long sessions because the browser animates the full distance. For very long sessions, the animation takes forever or the browser might give up mid-scroll.

2. `block: 'center'` - should be `block: 'start'` so the beginning of the message is at the top.

3. For long sessions with `contentVisibility: 'auto'`, the browser may not have accurate scroll positions because elements are lazy-rendered. This is why multiple clicks are needed - the first click triggers rendering of off-screen elements, which changes the scroll height, making the scroll position incorrect.


The fix is to switch to instant scrolling with `block: 'start'`, and add a retry mechanism with a short delay to account for the layout recalculation that happens when `contentVisibility: 'auto'` renders previously hidden elements. Now I'm implementing these changes.
</thinking>

# Experience: Hover Automation (Playwright + Electron)

## Проблема: Ложные пропуски событий (Stale Hover)
При тестировании интерактивных зон (например, Timeline Tooltip) стандартные команды перемещения мыши часто не вызывали срабатывание `onMouseEnter` или `onMouseLeave`. Тест видел элемент, но не мог вызвать его активацию.

## Причина: "Эффект Телепортации"
По умолчанию Playwright перемещает курсор из точки А в точку Б мгновенно. Если граница элемента (триггера) очень узкая, браузер может не зафиксировать пересечение границы курсором за один кадр. Это особенно критично для React-компонентов, которые ожидают цепочку событий.

## Решение: Плавная симуляция
Вместо мгновенного перемещения необходимо использовать параметр `steps` или циклы с микро-задержками:

```javascript
// ПЛОХО (мгновенно):
await page.mouse.move(100, 200);

// ХОРОШО (плавно):
await page.mouse.move(100, 200, { steps: 10 });
```

## Нюанс с Portals и relatedTarget
В автоматизированных тестах свойство `relatedTarget` при уходе мыши на портал (который находится вне иерархии триггера) почти всегда возвращает `null`. 
**Решение в коде:** Вместо проверки `e.relatedTarget` использовать проверку координат мыши (`mouseX`, `mouseY`) относительно границ элемента.
**Решение в тесте:** Двигать мышь за пределы "безопасной зоны" (мостика) медленно, чтобы дать JS-слушателям время отработать координаты.

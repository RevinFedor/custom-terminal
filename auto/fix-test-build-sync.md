# Experience: Test-Build Synchronization

## Проблема: Код в тесте не обновляется
После внесения правок в `src/main/main.js` или другие файлы основного процесса, тесты продолжают падать с той же ошибкой или ведут себя так, будто изменений не было.

## Причина: Работа с устаревшим /dist
Тестовый лаунчер запускает Electron-приложение, которое в режиме тестирования (или по умолчанию) обращается к скомпилированным файлам в папке `dist/`. 
В отличие от Renderer-процесса, который обновляется через Vite HMR, Main-процесс требует явной пересборки. `npm run dev` не всегда гарантирует мгновенную синхронизацию `dist/` для нужд внешнего тестового процесса.

## Решение: Принудительный билд
Перед запуском тестов, затрагивающих IPC-хендлеры или логику Main-процесса, необходимо выполнить ручную сборку:

```bash
npx electron-vite build
# или точечно для main
npx electron-vite build --outDir dist/main
```

## Урок для проекта
Если тест "не видит" ваш новый `console.log` или `ipcMain.handle` — проверьте дату изменения файлов в `dist/main/`. Скорее всего, вы тестируете старую версию приложения.

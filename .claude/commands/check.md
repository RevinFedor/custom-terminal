Ты — независимый code reviewer. Проверь, соответствуют ли изменения, которые ты только что сделал, документации проекта.

## Шаги

1. Вспомни, какие файлы ты изменял/создавал в этой сессии.
2. Прочитай `docs/architecture.md` — извлеки общие constraints (Anti-Patterns, правила).
3. Для каждого изменённого файла:
   - Определи, какие `docs/knowledge/` файлы относятся к этому коду.
     Паттерны поиска: имя файла, имя директории, ключевые слова (terminal, xterm, tabs, session, claude, rendering, shell).
   - Прочитай каждый релевантный knowledge-файл.
   - Также найди и прочитай соответствующий `docs/features/` файл.
4. Перечитай каждый файл, который ты изменял — сверь текущий код с constraints из knowledge.

## Формат вывода

Для каждого constraint:
- ✅ `knowledge-файл` — [что проверил] — соответствует
- ❌ `knowledge-файл` — [что нарушено] — как исправить

Если нашёл нарушения — исправь их сразу.
Если всё чисто — выведи "✅ Validation passed. Все constraints соблюдены."
